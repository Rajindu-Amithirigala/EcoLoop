Splash Screen

Technical Description

The Splash Screen, though built without the shared site template as the specs allowed, was crafted to align with the website's established visual identity. This meant adhering to the agreed-upon color palette, typography, and overall layout. A full-screen background image, fixed behind all other elements, was achieved with position: fixed and inset: 0. To enhance text readability against the image, an ::after (W3Schools, n.d.) pseudo-element overlay was implemented. A subtle pan animation, courtesy of @keyframes (W3Schools, n.d.) bgShift, was then applied. This animation shifts the background position over an 8-second interval, resulting in a soft movement effect. The background itself is intentionally sized at 105% by 105% to avoid any unsightly white edges that might appear at the animation's extremes.

The page features an automatic redirect, courtesy of a meta http-equiv="refresh" tag, set to trigger after four seconds, as outlined in the lecture notes. A CSS-only spinning loader (W3Schools, n.d.) was crafted, employing border-radius: 50% and animated, asymmetric border colors via @keyframes spin to create a fluid ring rotation. A progress bar was also included, animated with @keyframes shrink, which steadily decreased the fill width from 100% to 0% over a precise four-second interval, visually mirroring the countdown. The countdown number itself is refreshed every second, thanks to setInterval (W3Schools, n.d.) in JavaScript. This same script also enables the skip function, utilizing clearInterval to halt both the timer and the meta redirect when the user interacts with the page.

Accessibility

The background decorative layer is using aria-hidden="true" so that it is completely skipped by screen readers. The content wrapper is using role="main" so that the main content region is indicated. In the loader, the role="status" and aria-label are being used so that the screen readers can announce it correctly. In the paragraph of the countdown, aria-live="polite" is being used so that the changing number is announced to the screen reader users without interfering with the rest of the content. In the team members section, aria-label is being used on the container. The clamp() (W3Schools, n.d.) function has been used for the title and tagline font sizes so that the text can scale properly across different screen sizes without becoming unreadable on small screens.

Design and Implementation Rationale

The decision to use a full-screen background image with a dark overlay rather than a plain color background was made in order to provide a powerful visual first impression in accordance with the environmental theme of the site. The level of overlay opacity has been kept low enough so that the image is still visible while the white text above it remains readable. The spinning loader and shrinking progress bar were selected in order to provide the user with a clear indication that something is happening while they are waiting for the 4-second duration. This has the added benefit of reducing the chances of the user closing the page before the redirect can fire. The "skip" button has been added in order to honor the autonomy of the user and has been implemented using JavaScript in order to cancel the countdown interval. Pill-shaped tags have been used for the team member names in order to maintain a compact appearance while maintaining consistency with the other badge and button styles that are utilized throughout the rest of the site.

Action Impact Simulator

Technical Description

The Action Impact Simulator was created by importing the shared HTML template, including the website header, main content section, and footer, all of which are handled by components.js. The page's grid is defined by applying the .ais-layout class, which uses CSS Grid (W3Schools, n.d.) to divide the action cards and the impact results panel into two columns. This grid collapses to one column on screen sizes below 860px. The action cards, of which there are eight, are defined in HTML, each of which has a data attribute for points that is used by JavaScript to calculate the total score.

The card selection is controlled by JavaScript using a toggleCard() function that manipulates a class list and calls updateDisplay() to update the score. The level of impact is defined by a getLevel() function that has three defined thresholds: Low (1-6), Medium (7-13), and High (14-22), all stored in a LEVELS object that also has a label and message along with a class. Background image changes are driven by JavaScript by adding one of three CSS classes (ais-bg-low, ais-bg-medium, ais-bg-high) to the #bg-layer wrapper div, with the three background images defined in the embedded style block using background-attachment: fixed to create a parallax-style (W3Schools, n.d.) effect. A resetCards() function clears all selections and restores the default state. Keyboard support is implemented through an onkeydown (W3Schools, n.d.) handler that activates card selection on Enter or Space.

Accessibility

The action card grid has role="group" and aria-label attributes to identify it as a group of controls. Each card has role="checkbox" and aria-checked attributes so that screen readers can correctly announce the state of the selected card. Each card is keyboard accessible using tabindex="0" and responds to Enter and Space key (W3Schools, n.d.) presses. The impact results panel has aria-live="polite" attributes so that screen readers can correctly announce changes to the score and feedback messages without interrupting screen reader users. All thumbnail images have descriptive alt attributes. The skip link at the top of the page allows keyboard-only users to bypass the navigation links. The breadcrumb has aria-label and aria-current="page" attributes to correctly identify the user's location in the site.

Design and Implementation Rationale

The CSS Grid with auto-fill and minmax(260px, 1fr) (W3Schools, n.d.) was used for the action card grid so that it would automatically adjust to different screen sizes without needing to specify all column counts. For the impact panel, "sticky" positioning was used so that users could always see their current score when viewing the cards without needing to go back to the top of the page. For the background image system, JavaScript class switching was used instead of styles so that all of the visual styling is handled by CSS and all of the JavaScript logic is handled by JavaScript. backdrop-filter: blur (W3Schools, n.d.) was used on the card and panel backgrounds so that text remains readable when a background image is used.

Content Page

Technical Description

The shared HTML template and global CSS are used on the E-Waste content page, and an embedded style block defines page-specific styles. A sticky side navigation panel and the main article content are separated by a two-column structure created by CSS Grid in the main layout. In order to stay visible while scrolling without overlapping the fixed site header, the side navigation uses position: sticky (W3Schools, n.d.) with a top value determined from the --header-h CSS variable (W3Schools, n.d.). For the same purpose, scroll-margin-top is applied to each section heading to make sure anchor links are positioned correctly beneath the header.

The Introduction, Understanding the Problem, What Students Can Do, CircuitCycle, and Action Checklist are the five distinctly labeled sections that make up the page. The side navigation uses anchor hrefs to link to each of these sections. A two-column CSS Grid with aspect-ratio: 4/3 and object-fit: cover is used to display images in pairs, ensuring that thumbnails are always the same size regardless of the dimensions of the original image. The action checklist replaces the default list marker—which was eliminated by the global CSS reset—with a styled tick icon using a custom ::before pseudo-element (W3Schools, n.d.). To restore the numbering that the global reset eliminates, the statistics in the highlight box use a ol with list-style: decimal applied via embedded CSS. Only HTML and CSS are used in the implementation of the go-to-top button (W3Schools, n.d.).The go-to-top button is implemented using only HTML and CSS with position: fixed, as required by the specification.

Accessibility

The page has a labelled aside navigation with aria-label="Page sections, a breadcrumb with aria-current="page", and a skip link. For screen reader users, the checklist has an aria-label that provides context as a named list. Each of the four pictures has an alt text description that is pertinent to its content. Screen reader users can better comprehend the structure and navigate between sections thanks to the heading hierarchy, which follows a logical order from h1 down through h2 and h3 sub-sections. For clarity, the go-to-top button has title and aria-label attributes. In order to keep the content readable on mobile devices, the layout uses a media query (W3Schools, n.d.) to collapse to a single column on small screens. The Global E-Waste Monitor provided the statistics used on the page UNITAR (2024).

Design and Implementation Rationale

The reason for choosing this two-column format with a sticky side navigation is that this page has a lot of long-form content, and without proper internal navigation, a user would be forced to scroll extensively to find a particular section of content. The combination of aspect ratio and object fit was used on images to prevent layout shifts when images of different dimensions are placed side by side in the same grid row. For the highlight box, I used the main brand color for the background to ensure that the user would easily notice these important statistics, making it easier for them to find and recall this information within all that long-form content. For the tick styling in the checklist, I used pseudo-elements (W3Schools, n.d.) instead of images to ensure that it works in all browsers without having to add extra assets.

Challenges and Lessons Learned

Finding a balance between the Splash Screen's visual design and the technical specifications was one of its biggest challenges. It took careful coordination to implement the automatic redirect using a <meta http-equiv="refresh"> tag while simultaneously keeping a JavaScript-driven countdown and skip button so that both systems coexisted peacefully. While the meta refresh took care of the fallback redirect on its own, the skip button was required to cleanly terminate the JavaScript interval. In order to make sure that the text was readable over the background image on various screen sizes, I added a dark ::after overlay to the background layer and experimented with contrast at various opacity values. From this I learned that layered positioning and pseudo-elements are a reliable way to add visual depth without affecting the readability of the content above.

The JavaScript-driven background image system was one of the challenges that the Action Impact Simulator posed. It was necessary to carefully use background-attachment: fixed, semi-transparent card backgrounds and backdrop-filter: blur to make the background change dynamically based on score thresholds while maintaining consistently readable page content. It took multiple iterations to get the card selection logic, score calculation, progress bar update, and background class switching to all respond correctly from a single updateDisplay() function. I discovered that it is much simpler to debug and expand JavaScript code when it is organized around a central update function rather than dispersed individual handlers.

The biggest challenge for the Content Page was organizing a lot of information so that it felt manageable rather than overwhelming. In order to account for the fixed site header, which was not immediately apparent, the sticky side navigation must be implemented with the proper anchor positioning using scroll-margin-top on each section heading. Overall, working across these three pages deepened my understanding of how CSS, JavaScript and HTML work together as a system rather than independently, and reinforced the importance of planning the structure and interaction of a page before writing code.In order to restore numbering on the statistics ordered list, I also had to override the global CSS list reset. This experience taught me how embedded CSS can be used to selectively restore default browser behavior and how global resets can have unexpected effects on page-specific content.

Janet Compliance

The pages I implemented were created in accordance with responsible web publishing guidelines and Janet regulations (Jisc, 2025). Every piece of content on the Splash Screen, Action Impact Simulator, and Content Page is unique, educational, and pertinent to the coursework's educational goal. There was no offensive, defamatory, or deceptive content, and none of the pages were used for profit. In order to give the pages a clear, meaningful structure that is comprehensible to both browsers and assistive technologies, semantic HTML elements like <main>, <section>, <article>, <aside>, and <nav> were used throughout.

All three pages took accessibility into account. To make the pages usable by a variety of users, skip links, descriptive alt text, ARIA roles and labels, keyboard navigation support, and visible focus states were added. Aria-live regions are used in the countdown and score feedback on the Splash Screen and AIS, respectively, to notify screen reader users of dynamic content updates without requiring a page reload.
