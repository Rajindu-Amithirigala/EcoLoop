Feedback Page

Technical Description

The feedback page combines a scrollable programme directory, community ratings, and a structured feedback form. It acts as both a showcase of EcoLoop activity and a place where users can respond to the project.

A two-column CSS Grid separates the programme directory from the form on larger screens. The form uses <fieldset> and <legend> to group personal and feedback details semantically. A <datalist> supplies suggestions, while the programme directory uses a scrollable container with max-height and overflow-y: auto.

JavaScript provides a live character counter that updates on every input event. Form submission is intercepted, required fields are validated, the email field is checked with a regex pattern, and error styles are applied where needed. On success, the form is hidden and a confirmation message is shown.

Accessibility

  • Every form input has an associated <label> element for screen-reader compatibility.
  • Related controls are grouped using <fieldset> and <legend> for semantic clarity.
  • Validation error messages are displayed visually and exposed to assistive technologies.
  • Required fields are marked both visually (asterisk) and in markup (required attribute).
  • Colour contrast meets WCAG 2.1 AA standards across all form elements.

Design and Implementation Rationale

The two-column layout was chosen to allow users to browse programmes while simultaneously completing the feedback form. The scrollable directory panel prevents the page from becoming excessively long. Star ratings use Unicode characters and accent colours for a visually engaging yet lightweight display. Client-side validation provides immediate feedback without requiring a server round-trip, keeping the experience fast and self-contained.

Team Page

Technical Description

The team page introduces the four contributors using interactive profile cards. Each card reveals contribution details and biography text on hover or keyboard focus.

Cards are arranged in a four-column CSS Grid. Each card is keyboard-focusable via tabindex="0" and contains an initially collapsed detail area that expands using max-height and opacity transitions. Unique avatar gradients distinguish each member visually. Hover states apply a vertical lift with transform and an enhanced box-shadow.

Accessibility

  • Team cards are keyboard-focusable and respond to :focus-within states identically to hover.
  • Avatar images include descriptive alt text for screen readers.
  • The card list uses ARIA role="list" and role="listitem" for semantic structure.
  • Each card has an aria-label summarising the student's name and role.
  • Colour contrast between text and card backgrounds meets WCAG 2.1 AA requirements.

Design and Implementation Rationale

Interactive cards were chosen over a static table to make the team section more engaging. The expand-on-hover pattern keeps the page compact while allowing detailed information to be accessed on demand. The :focus-within CSS pseudo-class ensures keyboard users receive the same interactive experience as mouse users, meeting both accessibility and usability goals.

Fast Fashion Content (GoThrift 26')

Technical Description

The content_ST3.html page is titled GoThrift 26' and focuses on fast fashion — how cheap, trend-driven clothing affects the environment and how students can reduce clothing waste and shop more responsibly. It introduces the topic, explains the true cost (carbon, water, microplastics from synthetics), discusses student fashion habits including a “cost per wear” mindset, and outlines sustainable alternatives such as second-hand shopping, swaps, and capsule wardrobes.

The layout uses the shared sticky side navigation and article pattern. Content is structured into sections (Introduction, The True Cost, Student Fashion Habits, Sustainable Alternatives, Action Checklist, and the GoThrift 26' event). Imagery includes paired photos in a two-column img-row (for example fast_fashion.webp and clothing_pile.webp, plus impacts such as clothes_burn.jpg and water_pollution.jpg). A gradient stat-callout highlights key statistics; the action checklist uses a custom list with circular checkmark bullets via ::before; the closing event-section promotes the EcoLoop thrift-shop initiative.

Accessibility

  • All images include descriptive alt attributes for screen-reader users.
  • Semantic heading hierarchy (h1 through h3) provides clear document structure.
  • The stat callout presents full text for statistics, not colour alone; the checklist has an aria-label describing its purpose.
  • Internal links to related site pages (for example the Impact Simulator and food-waste content) use clear button labels.
  • Text contrast ratios meet WCAG 2.1 AA guidelines throughout the page.

Design and Implementation Rationale

The page is written for a UK student audience: local examples (charity shops, campus swaps, Vinted/Depop) keep the topic actionable. The stat callout separates headline figures from body copy; the checklist turns advice into concrete steps; the GoThrift event block gives a clear call toward community participation and aligns with the page title and breadcrumb (Fast Fashion). Together, these choices balance credibility, readability, and encouragement to change shopping habits without relying on a single warning style — the emphasis is on fashion waste, microplastics, and responsible consumption rather than electronic waste.

Challenges & Lessons

Challenge 1: Form Validation Without Libraries

Building validation from scratch required combining clear visual error states with accessible DOM feedback. The final solution used class changes for borders and helper elements for messages so users could understand problems quickly.

Challenge 2: Smooth Team Card Expansion

Animating height transitions required a practical fixed max-height rather than auto. Testing different values was necessary to find a balance that avoided clipping content while still feeling responsive.

Lessons Learned

  • Forms need testing with both keyboard and assistive-technology thinking, not just visually.
  • max-height transitions are a practical CSS solution for expandable panels.
  • Breaking large forms into semantic groups makes both the code and UX more manageable.
