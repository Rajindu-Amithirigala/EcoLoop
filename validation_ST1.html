Home Page validation report

The validation report of the Home Page did not find mistakes or warnings which verified that the HTML structure was programmed correctly and that the page corresponds to the main requirements of the W3C validator on the markup side. However, I realize that validation is a test of the markup correctness only, and it does not ensure the page is accessible and even well-designed. Due to this, I also took into account readability, keyboard navigation, alt text, the structure of the headings and the responsive behaviour, along with the validator results.

Gallery Page validation report

The gallery page validation report displayed no errors or warnings, which verified that the HTML structure of the page was correct despite the addition of higher level interactive elements like a thumbnail grid, modal dialog, buttons and ARIA attributes. This was significant as the gallery is based on structured HTML and JavaScript behaviour and passing the validation test ensured that the underlying markup was correct and standards-compliant.
