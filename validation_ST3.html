Feedback Page validation report

The validation report for the Feedback Page did not find mistakes or warnings, which verified that the HTML structure was programmed correctly and that the page corresponds to the main requirements of the W3C validator on the markup side. The page combines a programme directory, ratings, and a structured form using fieldsets, legends, labels, and HTML5 input types; passing validation confirmed that these elements were nested and associated correctly. However, I realize that validation is a test of the markup correctness only, and it does not ensure the page is accessible and even well-designed. Due to this, I also took into account readability, keyboard navigation, alt text, the structure of the headings and the responsive behaviour, along with the validator results.
