Content Page (Plastic Waste)

Technical Description

The plastic waste content page was created based off of a semantic structure that used <aside> and <article> elements to form a two-column content page. CSS Grid and sticky positioning used to make sure that the side navigation remains visible when browsing through the main article. Variables of style.css have been used to maintain the design integrity.

Accessibility

Keyboard accessibility was a priority whereby focus states are clearly visible. Images for plastic waste art texts include descriptive <alt> attributes and the aria labels are used for context of navigation links. Headings were arranged to be logically arranged to help the screen reader.

Design and Implementation Rationale

Semantic tags were used in a way to group up the content appropriately. The two-column arrangement ensures that the user can easily take advantage of the navigation menu without losing his place of course in the long line of the statistics and action items on plastic waste.

Sitemap Page

Technical Description

The Sitemap page was prepared by the interactive SVG format to visually map the structure of the website. The visual nodes are based on CSS transitions and not complicated JavaScript, to deliver hover effects: making the page lightweight and responsive

Accessibility

Focus states were added to the SVG elements to make keyboard users able to navigate through the map. Aria-labels were implemented to color legend so that the assistive technologies could rightly know how to announce the structure.

Design and Implementation Rationale

An SVG approach was chosen over a standard list to give a very visual and exciting representation of the site. Color coding of the nodes is correlated directly with the different sections: better usability

Profile Page

Technical Description

User Profile page using JavaScript to control a multiple step Interactive form. It accepts the user inputs and dynamically changes the progress bar according to the completion. CSS is used to take turns in showing the steps without reloading the page.

Accessibility

Form inputs have labeled correctly associated labels Error and success messages are dynamically injected and make use of their respective roles so that they can be announced by screen readers immediately. Keyboard navigability is enabled in all the interactive fields.

Design and Implementation Rationale

A step-by-step tabbed presentation was used so as not to overwhelm the user with the number of fields at a time. This logical progress, improves the overall user experience and prompts the user to complete the form.
