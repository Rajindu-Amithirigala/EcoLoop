Home page

Technical Description

The home page was built using a semantic structure with <main> wrapping around multiple <section> elements, <article> cards, headings, paragraphs, lists, and links (W3Schools, 2024). The hero area has a video background (Pennes, 2020) with a light overlay to make text readable. This was done using position: relative, position: absolute, z-index and ::before. Layout classes like container, section, card-grid, card, btn and badge from style.css (template) were integrated to keep the page looking consistent and on theme with the rest of the website. For the statistics strip and content cards I used CSS Grid, box-shadow, border-radius, object-fit, clamp(), and spacing variables from :root to create a nice layout (University of Westminster, 2026).

The cards under "Explore Topics Through Action" were designed with an interactive gallery using article elements, a clickable image area with descriptive alt text, CSS transitions, transforms and opacity changes. Additionally, JavaScript was used to add and remove a data-hovered attribute so that the hover effect is not purely CSS (W3Schools, 2019). This makes the page feel more interactive while following the specification guidelines.

Accessibility

I included two buttons in the hero section that link to the Impact Simulator and Food Waste page so that users can jump straight into the main content. The hero section also uses aria-labelledby, the background video includes <track> for validation requirements, and all images have descriptive <alt> text. The statistics strip is a labelled region and topic links use aria-label values to provide more context for screen reader users. All statistics included were drawn from UNITAR (2024), UNEP (2025), and Goodwin and Lipinski (2024).

I also made sure keyboard accessibility was considered. Focus styles are visible through :focus-visible. The topic cards respond to keyboard focus as well as mouse hover. The JavaScript also supports keyboard activation on the media links. Headings are used in a logical order which helps screen reader users understand the structure of the Home Page (Wang, 2024).

Design and Implementation Rationale

I selected semantic tags such as <main>, <section> and <article> on the Home Page so the content is grouped in a meaningful way for assistive technologies and easier to maintain. CSS Grid was used for the card areas because it gives a clean responsive structure without breaking reading order, while clamp() helps headings and spacing scale smoothly across different screen sizes. The video overlay, strong contrast and visible focus states were chosen to keep the hero section visually engaging without reducing readability or keyboard usability (Gittings, 2017).

Gallery Page

Technical Description

I implemented the Gallery as a JavaScript-driven image grid so the thumbnail content can be generated from an array named galleryData instead of repeating HTML manually. The page uses an unordered list as the gallery container and JavaScript creates each thumbnail as a <button> containing an <img> and a caption area (W3Schools, 2019). This approach makes the gallery easier to manage and update. The layout is controlled with CSS Grid through .thumb-grid and the thumbnail cards use aspect-ratio, object-fit, overflow, transform, and box-shadow to maintain a visual style (University of Westminster, 2026) and (W3Schools, 2019).

When a thumbnail is clicked, JavaScript opens a modal dialog that displays a larger image and description. The modal is built with a full-screen overlay, a content box, a close button and text customisation controls to change the font and color of the extended view. CSS uses position: fixed, inset, max-height, and responsive breakpoints to keep the modal responsive to different screen sizes. JavaScript controls the modal's opening, closing, focus handling, colour changes and font changes.

Accessibility

I used button elements for the thumbnails instead of plain div containers so they are naturally keyboard accessible. Each thumbnail has attributes to describe its function. The modal uses role="dialog" and other attributes so assistive technologies can understand it as a focused interactive window. The close button also has an aria-label.

Keyboard support was added for the modal by allowing the Escape key to close it and by trapping focus inside the dialog while it's open. When the modal closes, focus returns to the thumbnail that opened it. All gallery images use alt text and the customisation buttons use aria-pressed so their selected state is announced properly. All gallery images were sourced from the United Nations SDG: 12 photo library (UN Photo, 2020). These images are directly related to Responsible Consumption and Production. (United Nations, 2025).

Design and Implementation Rationale

I chose button-based thumbnails instead of generic containers because the gallery is an interactive control and should be reachable and understandable through keyboard navigation by default (GeeksforGeeks, 2024). CSS Grid and aspect-ratio were used to keep the thumbnails aligned and predictable, which improves scanning and reduces visual clutter. The modal structure, overlay and focus handling were implemented to keep attention on the selected item, while aria labels and state indicators make the interaction clearer for screen reader users.

Content Page

Technical Description

I structured the ST1 content page using a shared page heading followed by a two-column content layout. The layout combines an aside for navigation and an article for the main reading content. I divided the page into thematic sections with unique IDs which make the content easier to scan and also support in-page navigation. CSS Grid is used to separate the side navigation from the article and position: sticky keeps the side navigation visible while scrolling on screens. (University of Westminster, 2026) and (W3Schools, 2019)

The page styling focuses on readability and content hierarchy (Haim, 2025). I used styles to place related images side by side, highlight statistics, and create visual action points. I also used scroll-margin-top on headings so anchor links do not get hidden behind the site header. Shared styles help keep the page consistent with the rest of the website.

Accessibility

The page includes a skip link, a breadcrumb with aria-current="page" and a labelled side navigation with aria-label. The internal section links make long content easier to navigate for both keyboard users and screen reader users. Because each section has a heading and ID, users can move through the content in a meaningful order and jump directly to the part they need. Content page images were sourced from Freepik (Freepik.com, 2026). The statistics were sourced from the World Food Programme (2024), the FAO Regional Office (2022), and UN DESA Voice (2023).

All main images contain descriptive <alt> text and the checklist points are labelled to provide better context for assistive technologies. The layout is responsive so it changes to a single-column structure on smaller screen sizes. The back-to-top button has a simple up arrow icon and helps users navigate the lengthy page easily.

Design and Implementation Rationale

I used an aside for local navigation and an article for the main reading area so the long-form content has a clear structural separation between navigation and information. CSS Grid was the best choice for the two-column layout because it keeps the side menu and article aligned while still allowing the design to collapse neatly on smaller screens. Sticky positioning, section IDs and scroll-margin-top were chosen to support quick movement through the page and to ensure anchor navigation remains usable when a fixed or sticky header is present.

Challenges and Lessons Learned

One of the main challenges I had on the Home Page was to come up with a visually strong hero section while keeping the text readable and the layout responsive. The background video, the overlay and foreground information need to work with one another without diminishing the contrast or crowding the page. I solved this problem with some use of layered positioning, a dark overlay, controlling content width and responsive font sizing. The challenge of giving an interactive feel to the topic cards without losing keyboard accessibility was another issue. I was able to solve this by using a combination of CSS hover, state changes using JavaScript and visible focus styling. From this I learnt that good visual design must always have a balance of readability, structure and accessibility.

On the Gallery Page, the most difficult thing to do was to construct the modal interaction in a way that is both dynamic and usable. I created the thumbnail grid with the help of JavaScript, opening the item the user clicked on in an extended modal view, updating the image and text content and also allowing the user to customise the text styling. The modal state, close behaviour, keyboard navigation and focus return were the biggest challenges. I was able to resolve this problem by using a structured array to store the gallery data and developing separate functions to open, close and reset the modal. Working on this page taught me the importance of organising JavaScript in a clear way when a page has several features that are interactive, and showed me the need to think about what needs to be done to make the page accessible in addition to making it functional.

For the Content Page, the major challenge was to convey a large amount of information in a manner that was still easy to read and navigate. Long pages might easily become daunting if the design is dull or the navigation from one part to another is not easily facilitated for users. I skirted this issue by implementing a side navigation menu using anchor links, clear section headings, supporting images and a highlighted statistics box and an action checklist to break up the content. I added scroll-margin-top and a sticky side menu as well because the internal links will be functional with the fixed site header. From this, I learned that content design was not just writing information, but also structuring information in a manner that the user is able to move through efficiently.

Overall, these pages helped me in enhancing my knowledge about responsive design, reusable styling, and interactive JavaScript. The main lesson I learned was that good web design is not only about looks. It is dependent on having clear structure, accessibility, maintainability and ensuring that each feature functions well for all types of different users and devices.

Janet Compliance

The pages I created were designed to follow Janet regulations (Jisc, 2025) and general good practice for public web publication, keeping the content lawful, respectful, accurate and suitable for an educational website. I did not include defamatory, offensive or misleading material and the purpose of the pages is informational rather than harmful or commercial. I also made use of the semantic HTML elements such as headings, sections, navigation, forms and lists so that the structure of the pages was obvious and accessible. Throughout my pages I included such things as skip links, descriptive alt text, labelled navigation and keyboard-friendly controls and visible focus states; features which help with accessible publication and responsible web design. I also took into consideration copyright and data protection. Any third-party images, video or external material used on the pages were properly acknowledged in the references section and I have tried to use content in such a way that it is acceptable for the coursework rather than claiming the resources of an external source.

Overall, I believe that my pages are compliant as far as structure, accessibility and responsible publishing practice are concerned. However, to be fully realistic, another important area to check is that all the external media, all borrowed factual descriptions are referenced clearly and consistently. Compliance is therefore not only about writing code that works, but also writing content in an ethical, legal and accessible manner.
